# Windows 2.1.0 查询页统计与状态图表

## 背景

旧查询页只统计 `使用中`、`提醒未休息`、`休息中` 三种状态，并且表格只显示绝对时长。曲线图固定使用日志中的数值：`使用中 y=1`、`提醒未休息 y=0.3`、`休息中 y=0`。其中 `提醒未休息 y=0.3` 在视觉上过于靠近休息状态，不利于表达“仍然接近用眼中”的语义。

本次升级把查询页从“原始日志绘图”改为“语义状态展示”：日志值先映射为状态，再由界面配置决定如何统计和展示。

## 实现思路

- 不批量修改历史日志文件，避免破坏原始数据。
- 在读取日志时建立兼容映射：
  - `1.0` -> `使用中`
  - `0.3` -> `提醒未休息`
  - `0.7` -> `提醒未休息`
  - `0.0` -> `休息中`
  - `-1.0` -> `未记录`
- 查询页内部使用状态 key 处理数据，而不是直接依赖日志里的 y 值。
- 图表的 y 值变成显示配置，默认值为：
  - `使用中`: `1.0`
  - `提醒未休息`: `0.7`
  - `休息中`: `0.0`
  - `未记录`: `-0.1`
- 对于查询区间开始前没有任何已知状态的时间段，归入 `未记录`。
- 对于明确写入 `-1.0` 的锁屏、休眠、退出或离线区段，归入 `未记录`。
- 统计表以查询区间总时长为分母，展示 `绝对时长 + 百分比`。
- 显示百分比时进行一位小数的尾差修正，让四种状态显示值合计为 `100.0%`。

## 修改内容

- 新增 `未记录` 状态
  - 用于表示锁屏、休眠、程序退出、无已知记录等时间。
  - 表格新增 `未记录(y=-0.1)` 列。
  - 图表新增灰色展示。

- 统计表升级
  - 原表格单元格从 `HH:MM:SS` 改为 `HH:MM:SS (xx.x%)`。
  - 状态列从 3 列扩展到 4 列。
  - 四种状态的比例合计为 `100.0%`。

- 图表设置接口
  - 在 `状态曲线` 栏目加入 `呈现方式` 下拉框。
  - 支持 `时间轴色块` 和 `曲线图` 两种模式。
  - 为四种状态加入 y 值输入框。
  - 提供 `重置显示` 按钮恢复默认配置。
  - 配置保存到 `usage_query_settings.json`，不会影响主程序计时设置。

- 曲线图升级
  - 不再直接使用日志里的 `0.3` 作为展示位置。
  - 根据当前 y 值配置绘制状态阶梯线。
  - `未记录` 使用灰色虚线。

- 时间轴色块
  - 使用红色块表示 `使用中`。
  - 使用橙色块表示 `提醒未休息`。
  - 使用青绿色块表示 `休息中`。
  - 使用灰色块表示 `未记录`。

- 查询条件微调
  - 将自定义时间范围的确认按钮从 `Apply` 改为 `应用`，保持查询页文案一致。

## 旧日志处理方案

本次不修改已有 `usage_logs/*.txt` 文件。原因是旧日志是原始行为记录，直接改写会带来回滚困难和数据污染风险。

兼容由读取层完成：

- 旧记录 `0.3` 继续有效，语义为 `提醒未休息`。
- 如果未来某个版本开始写入 `0.7`，也会被识别为同一个 `提醒未休息` 状态。
- 图表显示位置由查询页配置决定，与日志原始值解耦。

这个方案既保留历史数据，又允许界面默认把 `提醒未休息` 放到更合理的 `y=0.7`。

## 验证

- 使用 `python -m py_compile` 编译 Windows 端 Python 源文件。
- 使用单元式脚本验证：
  - `0.3` 和 `0.7` 都映射为 `提醒未休息`。
  - `-1.0` 映射为 `未记录`。
  - 四种状态统计时长正确。
  - 表格百分比合计为 `100.0%`。
- 使用 Qt offscreen 脚本验证：
  - 查询页表格新增 `未记录` 列。
  - 默认 `提醒未休息 y=0.7`。
  - 默认呈现方式为 `时间轴色块`。
  - 可切换为 `曲线图`。
  - y 值输入可以更新图表配置。
- 使用 PyInstaller 构建 Windows onedir 程序。
- 使用 Inno Setup 生成 `Triple2EyeProtection-Setup-2.1.0.exe`。
